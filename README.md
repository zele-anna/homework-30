# Платформа для онлайн-обучения
## Цель проекта
Разработка приложения для онлайн-обучения

### Установка и настройка
Все необходимые зависимости указаны в файле [pyproject.toml](pyproject.toml)
Для работы необходимо создать базу данных PostgreSQL и прописать параметры доступа в базу в файле .env по примеру [.env.example](.env.example)

### Запуск программы
Требуется запустить сервер командой python manage.py runserver

### Использование функций и примеры работы
Все эндпоинты закрыты для неавторизованных пользователей, исключения - эндпоинты для регистрации пользователей и входа.
1. Регистрация пользователей производится по модели User, содержащей поля:
- email (логин);
- телефон; 
- город; 
- аватарка.
2. Зарегистрированный пользователь имеет возможность создавать курсы. Модель Course содержит поля:
- название;
- превью (картинка); 
- описание.

При создании курса осуществляется привязка его к текущему пользователю в качестве владельца.
После создания курса владелец имеет права на CRUD операции по нему.
Пользователь имеет права на работу только со своими курсами.
Эндпоинты работы с курсами реализованы через Viewsets.
Для курса имеется единственный сериализатор CourseSerializer, в котором также имеются поля для вывода:
- количества уроков в данном курсе;
- списка уроков в данном курсе; 
- подписки на обновления курса (см. п. 4).

3. Зарегистрированный пользователь имеет возможность создавать уроки. Модель Lesson содержит поля:
- название;
- превью (картинка); 
- описание; 
- ссылка на видео с хостинга youtube.com (имеется валидация ссылки по данному хостингу);
- курс (привязка через FK).

При создании урока осуществляется привязка его к текущему пользователю в качестве владельца.
После создания урока владелец имеет права на CRUD операции по нему.
Пользователь имеет права на работу только со своими уроками.
Эндпоинты работы с уроками реализованы через Generic-классы.
Для урока реализовано два сериализатора:
- LessonSerializer для сериализации всех полей модели урока;
- LessonShortSerializer - для вывода сокращенного списка полей в списке уроков по курсу.

4. Подписка реализована в качестве класса Subscription.
Создание объекта класса производится при обращении к эндпоинту подписки в сериализаторе курса.
Если подписка оформлена, при повторном обращении объект подписки удаляется.
5. Реализована модель оплат Payment с полями:
- пользователь, оплативший курс/урок; 
- дата оплаты (текущие дата и время); 
- оплаченный курс или урок (привязка по FK к модели Курса или Урока); 
- сумма оплаты; 
- способ оплаты: наличные или перевод на счет.

Настроена фильтрация для эндпоинта вывода списка платежей с возможностями:
- менять порядок сортировки по дате оплаты;
- фильтровать по курсу или уроку;
- фильтровать по способу оплаты.

6. Имеется возможность создания группы модераторов. Модераторы имеют права:
- работы с любыми курсами, но без возможности их удалять и создавать новые.
- работы с любыми уроками, но без возможности их удалять и создавать новые.

Заполнение группы модераторов с правами на просмотр и редактирование курсов и уроков можно выполнить с помощью фикстуры [groups.json](users/fixtures/groups.json)

7. Для вывода списка курсов и уроков используется пагинация CustomPagination.
